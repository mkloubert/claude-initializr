Μια διαδικτυακή εφαρμογή για τη δημιουργία αρχείων ρύθμισης Docker για την ασφαλή εκτέλεση του [Claude Code](https://docs.anthropic.com/en/docs/claude-code) σε περιβάλλον container.

**Ζωντανή Επίδειξη:** [https://claude.kloubert.dev](https://claude.kloubert.dev)

## Χαρακτηριστικά

### Ρύθμιση Dockerfile

- **Βασική Εικόνα**: Ρυθμίστε το όνομα και την έκδοση της βασικής εικόνας Docker (προεπιλογή: `node:24`)
- **Επιλογή Λογισμικού**: Επιλέξτε επιπλέον λογισμικό για εγκατάσταση:
  - ffmpeg (επεξεργασία ήχου/βίντεο)
  - Flutter (περιλαμβάνει Dart και Android SDK)
  - Go
  - ImageMagick (επεξεργασία εικόνας)
  - Python 3
  - Rust (περιλαμβάνει τον διαχειριστή πακέτων Cargo)
  - TypeScript
  - uv (γρήγορος εγκαταστάτης πακέτων Python, συνιστά Python)
- **Ρύθμιση Εκδόσεων**: Οι εκδόσεις λογισμικού ρυθμίζονται μέσω παραμέτρων build του Docker (π.χ. `--build-arg GO_VERSION=1.22.0`)
- **Προσαρμοσμένα Πακέτα APT**: Προσθέστε επιπλέον πακέτα Debian/Ubuntu για εγκατάσταση στο container
- **Προσαρμοσμένα Πακέτα NPM**: Προσθέστε επιπλέον πακέτα NPM για καθολική εγκατάσταση, με την επιλογή εγκατάστασης ως χρήστης `root` ή `node`
- **Προσαρμοσμένες Εντολές RUN**: Προσθέστε προσαρμοσμένες εντολές shell για εκτέλεση κατά τη δημιουργία της εικόνας Docker, με την επιλογή εκτέλεσης ως χρήστης `root` ή `node`

### Ρύθμιση docker-compose.yaml

- **Μεταβλητές Περιβάλλοντος**: Ρυθμίστε μεταβλητές περιβάλλοντος για το αρχείο `.env` σας
- **Προστατευμένα Αρχεία**: Καθορίστε αρχεία που πρέπει να προστατεύονται με την τοποθέτηση κενών αρχείων μόνο για ανάγνωση (αποτρέπει την πρόσβαση σε ευαίσθητα αρχεία όπως `.env.local`)

### Επεξεργαστής CLAUDE.md

- Επεξεργαστής Markdown με επισήμανση σύνταξης
- Ενσωματωμένη λειτουργία προεπισκόπησης
- Γράψτε οδηγίες ειδικές για το έργο για τον Claude

### Διαμόρφωση settings.json

- **Κανόνες δικαιωμάτων**: Διαμορφώστε τα δικαιώματα του Claude Code για έλεγχο πρόσβασης αρχείων
  - `Allow` - Κανόνες για αυτόματα επιτρεπόμενες λειτουργίες
  - `Ask` - Κανόνες που απαιτούν επιβεβαίωση χρήστη
  - `Deny` - Κανόνες που πάντα απορρίπτονται
- **Υποστηριζόμενες οδηγίες**:
  - `Read()` - Ελέγχει ποια αρχεία μπορεί να διαβάσει ο Claude (π.χ: `Read(src/**)`)
  - `Edit()` - Ελέγχει ποια αρχεία μπορεί να τροποποιήσει ο Claude (π.χ: `Edit(.env)`)
  - `WebFetch()` - Ελέγχει την πρόσβαση δικτύου (π.χ: `WebFetch(https://api.github.com:*)`)
- **Αυτόματη ενσωμάτωση**: Τα προστατευμένα αρχεία προστίθενται αυτόματα ως κανόνες απόρριψης `Read()`
- **Υποστήριξη μοτίβων Glob**: Χρησιμοποιήστε μοτίβα όπως `src/**` για αναδρομική αντιστοίχιση

### Διαμόρφωση DevContainer (VS Code / GitHub Codespaces)

- **Ενσωμάτωση VS Code**: Δημιουργήστε `devcontainer.json` για VS Code Dev Containers
- **GitHub Codespaces**: Συμβατή διαμόρφωση για ανάπτυξη στο GitHub Codespaces
- **Επεκτάσεις**: Διαμορφώστε επεκτάσεις VS Code για αυτόματη εγκατάσταση
- **Ρυθμίσεις**: Ορίστε ρυθμίσεις VS Code για το περιβάλλον του κοντέινερ
- **Features**: Προσθέστε Dev Container Features (π.χ. GitHub CLI, πρόσθετες γλώσσες)
- **Προώθηση θυρών**: Διαμορφώστε θύρες για προώθηση από το κοντέινερ
- **Εντολές κύκλου ζωής**: Ρυθμίστε εντολές για συμβάντα post-create, post-start και post-attach
- **Προτεινόμενες επεκτάσεις**: Αυτόματες προτάσεις επεκτάσεων με βάση το επιλεγμένο λογισμικό

### Σύγχρονη Διεπαφή

| Συντόμευση | Ενέργεια |
| ---------- | -------- |
| `Ctrl/⌘ + S` | Λήψη ZIP |
| `Ctrl/⌘ + B` | Εναλλαγή πλευρικής μπάρας |
| `Ctrl/⌘ + E` | Εναλλαγή παράθυρου προεπισκόπησης |
| `Ctrl/⌘ + Z` | Αναίρεση |
| `Ctrl/⌘ + Y` | Επανάληψη |
| `Ctrl/⌘ + Shift + Z` | Επανάληψη (εναλλακτικό) |
| `Ctrl/⌘ + Shift + D` | Εναλλαγή σκοτεινού/φωτεινού θέματος |
| `Ctrl/⌘ + Shift + X` | Επαναφορά στις προεπιλογές |
| `Ctrl/⌘ + Shift + L` | Άνοιγμα επιλογέα γλώσσας |

Ανοίξτε τη βοήθεια συντομεύσεων πληκτρολογίου μέσω του μενού ρυθμίσεων στην κεφαλίδα.

### Μηχανισμός Αυτόματης Αποθήκευσης

Η λειτουργία αυτόματης αποθήκευσης μπορεί να ενεργοποιηθεί/απενεργοποιηθεί χρησιμοποιώντας το εικονίδιο αποθήκευσης στην κεφαλίδα:

| Εικονίδιο          | Κατάσταση        | Συμπεριφορά                                                             |
| ------------------ | ---------------- | ----------------------------------------------------------------------- |
| 💾 (Αποθήκευση)    | Ενεργοποιημένο   | Όλες οι αλλαγές αποθηκεύονται αυτόματα στο localStorage                 |
| 🚫💾 (Απενεργ.)    | Απενεργοποιημένο | Οι αλλαγές δεν αποθηκεύονται· τα υπάρχοντα δεδομένα διαγράφονται        |

**Πώς λειτουργεί:**

- **Ενεργοποίηση αυτόματης αποθήκευσης**: Αποθηκεύει αμέσως τις τρέχουσες ρυθμίσεις στο localStorage
- **Απενεργοποίηση αυτόματης αποθήκευσης**: Διαγράφει όλες τις αποθηκευμένες ρυθμίσεις από το localStorage
- Η προτίμησή σας για αυτόματη αποθήκευση διατηρείται μεταξύ συνεδριών

### Εισαγωγή/Εξαγωγή ρυθμίσεων

Μπορείτε να μοιραστείτε ή να δημιουργήσετε αντίγραφα ασφαλείας των ρυθμίσεών σας μέσω αρχείων JSON:

- **Εξαγωγή**: Κάντε κλικ στο εικονίδιο μεταφόρτωσης στην κεφαλίδα για να κατεβάσετε τις τρέχουσες ρυθμίσεις ως `claude-initializr-config.json`
- **Εισαγωγή**: Κάντε κλικ στο εικονίδιο λήψης για να επιλέξετε ένα αρχείο JSON που είχε εξαχθεί προηγουμένως

**Πώς λειτουργεί:**

- Η **εξαγωγή** αποθηκεύει όλες τις ρυθμίσεις (βασική εικόνα, επιλογή λογισμικού, πακέτα, εντολές, δικαιώματα, περιεχόμενο CLAUDE.md) σε ένα μόνο αρχείο JSON
- Η **εισαγωγή** επικυρώνει το αρχείο, εμφανίζει μια προεπισκόπηση των αλλαγών και ζητά επιβεβαίωση πριν από την εφαρμογή
- Για λόγους ασφαλείας, οι **τιμές των μεταβλητών περιβάλλοντος δεν περιλαμβάνονται ποτέ** στα εξαγόμενα αρχεία — εξάγονται μόνο τα ονόματα των μεταβλητών
- Οι εισαγόμενες ρυθμίσεις λαμβάνουν νέα εσωτερικά αναγνωριστικά για την αποφυγή συγκρούσεων
- Η μορφή εξαγωγής περιλαμβάνει ένα πεδίο έκδοσης (`"version": "1.0"`) για συμβατότητα προς τα εμπρός

### Απόρρητο & Αποθήκευση Δεδομένων

Αυτή η εφαρμογή σέβεται το απόρρητό σας:

- **Μόνο Τοπική Αποθήκευση**: Όλες οι ρυθμίσεις αποθηκεύονται τοπικά στο πρόγραμμα περιήγησής σας (localStorage)
- **Καμία Επικοινωνία με Διακομιστή**: Κανένα δεδομένο δεν αποστέλλεται ποτέ σε κανέναν διακομιστή
- **Ασφαλές από Σχεδιασμό**: Οι **τιμές** των μεταβλητών περιβάλλοντος **δεν αποθηκεύονται ποτέ** - αποθηκεύονται μόνο τα ονόματα των μεταβλητών
- **Πλήρης Έλεγχος**: Μπορείτε να απενεργοποιήσετε την αυτόματη αποθήκευση ανά πάσα στιγμή χρησιμοποιώντας τον διακόπτη στην κεφαλίδα, ο οποίος επίσης διαγράφει όλα τα αποθηκευμένα δεδομένα
- **Θέμα με βάση τη Συνεδρία**: Η προτίμηση θέματος επαναφέρεται στην προεπιλογή του συστήματος κατά την επαναφόρτωση της σελίδας

## Χαρακτηριστικά Ασφαλείας

Η δημιουργημένη ρύθμιση Docker περιλαμβάνει ολοκληρωμένα μέτρα ασφαλείας:

### Τείχος Προστασίας Δικτύου

Το σενάριο `init-firewall.sh` υλοποιεί αυστηρή απομόνωση δικτύου:

- **Τείχος προστασίας βασισμένο σε iptables** με πολιτική DROP για όλη την εξερχόμενη κίνηση
- **Προσέγγιση μόνο με λίστα επιτρεπόμενων** - μόνο οι εγκεκριμένοι τομείς είναι προσβάσιμοι:
  - `api.anthropic.com` - Claude API
  - `npm registry` - Εγκατάσταση πακέτων
  - `github.com` - Λειτουργίες Git
  - `sentry.io` - Αναφορά σφαλμάτων
- **Αυτόματη επίλυση IP GitHub** για web, API και git endpoints
- **Απομόνωση δικτύου host** - αποτρέπει την πρόσβαση στο τοπικό δίκτυο
- **Επαλήθευση τείχους προστασίας** - οι δοκιμές διασφαλίζουν ότι οι κανόνες εφαρμόζονται σωστά

### Ενίσχυση Ασφάλειας Docker

- **Απόρριψη ικανοτήτων**: Όλες οι ικανότητες Linux αφαιρούνται (`cap_drop: ALL`)
- **Χωρίς κλιμάκωση προνομίων**: `no-new-privileges:true`
- **Όρια πόρων**: Περιορισμοί CPU και μνήμης
- **Τοποθετήσεις μόνο για ανάγνωση**: Τα προστατευμένα αρχεία τοποθετούνται ως μόνο για ανάγνωση
- **Μη-root εκτέλεση**: Εκτελείται ως χρήστης `node`

## Χρήση

1. **Ρύθμιση Βασικής Εικόνας**: Ορίστε το όνομα και την έκδοση της βασικής εικόνας Docker (π.χ., `node:24` ή `node:22-slim`)

2. **Επιλογή Λογισμικού**: Επιλέξτε ποιο επιπλέον λογισμικό θα εγκατασταθεί στο container σας

3. **Προσθήκη Προσαρμοσμένων Πακέτων και Εντολών**:
   - Προσθέστε προσαρμοσμένα πακέτα APT (π.χ., `curl`, `graphviz`, `sqlite3`)
   - Προσθέστε προσαρμοσμένα πακέτα NPM για καθολική εγκατάσταση (π.χ., `eslint`, `prettier`)
   - Επιλέξτε αν τα πακέτα NPM πρέπει να εγκατασταθούν ως χρήστης `node` (προεπιλογή) ή `root`
   - Προσθέστε προσαρμοσμένες εντολές RUN για εκτέλεση κατά τη δημιουργία (π.χ., `pip install numpy`)
   - Επιλέξτε αν οι εντολές RUN πρέπει να εκτελούνται ως χρήστης `node` (προεπιλογή) ή `root`

4. **Ορισμός Μεταβλητών Περιβάλλοντος**: Προσθέστε οποιεσδήποτε μεταβλητές περιβάλλοντος χρειάζεται το έργο σας (π.χ., `ANTHROPIC_API_KEY`)

5. **Προστασία Ευαίσθητων Αρχείων**: Προσθέστε διαδρομές σε αρχεία που πρέπει να προστατεύονται (π.χ., `.env.local`)

6. **Επεξεργασία CLAUDE.md**: Γράψτε οδηγίες για τον Claude στον επεξεργαστή Markdown

7. **Διαμόρφωση δικαιωμάτων**: Ρυθμίστε κανόνες δικαιωμάτων στην κάρτα settings.json
   - Προσθέστε κανόνες `Allow` για αυτόματα εγκεκριμένες λειτουργίες
   - Προσθέστε κανόνες `Ask` για λειτουργίες που απαιτούν επιβεβαίωση
   - Προσθέστε κανόνες `Deny` για απαγορευμένες λειτουργίες
   - Τα προστατευμένα αρχεία προστίθενται αυτόματα ως κανόνες απόρριψης `Read()`

8. **Προεπισκόπηση**: Ελέγξτε τα δημιουργημένα αρχεία ρύθμισης στις καρτέλες προεπισκόπησης

9. **Λήψη**: Κάντε κλικ στο "Λήψη ZIP" για να λάβετε όλα τα αρχεία

## Χρήση των Δημιουργημένων Αρχείων

1. Εξαγάγετε το αρχείο ZIP στον κατάλογο του έργου σας

2. Αντιγράψτε τα αρχεία του έργου σας στον φάκελο `workspace` (ή τοποθετήστε το υπάρχον έργο σας)

3. Ορίστε το κλειδί API σας στο αρχείο `.env`:

   ```bash
   ANTHROPIC_API_KEY=το-κλειδί-api-σας-εδώ
   ```

4. Κατασκευάστε και εκτελέστε το container:

   ```bash
   docker compose up --build
   ```

   **Προαιρετικό: Προσαρμοσμένες εκδόσεις λογισμικού**

   Μπορείτε να προσαρμόσετε τις εκδόσεις λογισμικού κατά τη δημιουργία του Docker χρησιμοποιώντας παραμέτρους build:

   ```bash
   docker compose build \
     --build-arg CLAUDE_CODE_VERSION=1.0.3 \
     --build-arg FLUTTER_VERSION=3.29.3 \
     --build-arg GIT_DELTA_VERSION=0.18.2 \
     --build-arg GO_VERSION=1.24.3 \
     --build-arg PYTHON_VERSION=3.13.2 \
     --build-arg TYPESCRIPT_VERSION=5.8.3 \
     --build-arg ZSH_IN_DOCKER_VERSION=1.2.0
   ```

   | Παράμετρος build | Προεπιλογή | Περιγραφή |
   |------------------|------------|-----------|
   | `CLAUDE_CODE_VERSION` | `stable` | Έκδοση Claude Code (`latest` ή συγκεκριμένη όπως `1.0.58`) |
   | `FLUTTER_VERSION` | `latest` | Έκδοση Flutter SDK |
   | `GIT_DELTA_VERSION` | `0.18.2` | Έκδοση git-delta |
   | `GO_VERSION` | `latest` | Έκδοση Go |
   | `PYTHON_VERSION` | `latest` | Έκδοση Python 3 |
   | `TYPESCRIPT_VERSION` | `latest` | Έκδοση TypeScript |
   | `ZSH_IN_DOCKER_VERSION` | `1.2.0` | Έκδοση zsh-in-docker |

   **Προαιρετικό: Προσαρμοσμένες διευθύνσεις URL λήψης**

   Αν χρειάζεστε mirror ή proxy για τις λήψεις πακέτων, μπορείτε να αντικαταστήσετε τις προεπιλεγμένες διευθύνσεις URL κατά τη δημιουργία. Όλες οι διευθύνσεις URL υποστηρίζουν παραμέτρους ερωτήματος:

   ```bash
   docker compose build \
     --build-arg GO_JSON_URL=https://my-mirror.example.com/golang/?mode=json \
     --build-arg GO_DOWNLOAD_URL=https://my-mirror.example.com/golang \
     --build-arg RUSTUP_INSTALL_URL=https://my-mirror.example.com/rustup/rustup-init.sh \
     --build-arg FLUTTER_JSON_URL=https://my-mirror.example.com/flutter/releases_linux.json \
     --build-arg FLUTTER_BASE_URL=https://my-mirror.example.com/flutter/releases \
     --build-arg UV_INSTALL_SCRIPT_URL=https://my-mirror.example.com/uv/install.sh
   ```

   | Παράμετρος build | Προεπιλογή | Περιγραφή |
   |------------------|------------|-----------|
   | `GO_JSON_URL` | `https://go.dev/dl/?mode=json` | URL για το JSON API εκδόσεων Go (μόνο για "latest") |
   | `GO_DOWNLOAD_URL` | `https://go.dev/dl` | Βασική URL για λήψεις αρχείων Go |
   | `RUSTUP_INSTALL_URL` | `https://sh.rustup.rs` | URL για το σενάριο εγκατάστασης rustup |
   | `FLUTTER_JSON_URL` | `https://storage.googleapis.com/flutter_infra_release/releases/releases_linux.json` | URL για το JSON API εκδόσεων Flutter (μόνο για "latest") |
   | `FLUTTER_BASE_URL` | `https://storage.googleapis.com/flutter_infra_release/releases` | Βασική URL για λήψεις αρχείων Flutter |
   | `UV_INSTALL_SCRIPT_URL` | `https://astral.sh/uv/install.sh` | URL για το σενάριο εγκατάστασης uv |

5. Συνδεθείτε στο container:

   ```bash
   docker compose exec claude zsh
   ```

6. Αρχικοποιήστε το τείχος προστασίας (απαιτεί κωδικό sudo):

   ```bash
   sudo /usr/local/bin/init-firewall.sh
   ```

7. Εκκινήστε το Claude Code:
   ```bash
   claude
   ```

## Αντιμετώπιση Προβλημάτων

### Προβλήματα Τείχους Προστασίας

Αν αντιμετωπίσετε προβλήματα δικτύου μετά την ενεργοποίηση του τείχους προστασίας:

```bash
# Έλεγχος κατάστασης τείχους προστασίας
sudo iptables -L -n

# Προβολή αποκλεισμένων συνδέσεων
sudo iptables -L -n -v | grep DROP

# Επαναφορά τείχους προστασίας (επιτρέπει όλη την κίνηση)
sudo iptables -F
```

### Το Container δεν Ξεκινά

```bash
# Έλεγχος logs
docker compose logs

# Ανακατασκευή χωρίς cache
docker compose build --no-cache
```

### Άρνηση Άδειας

Βεβαιωθείτε ότι ο κατάλογος workspace έχει τις σωστές άδειες:

```bash
chmod -R 755 workspace
```

### Επαναφορά Ρυθμίσεων Εφαρμογής

Για να διαγράψετε όλες τις αποθηκευμένες ρυθμίσεις και να ξεκινήσετε από την αρχή, ανοίξτε την κονσόλα προγραμματιστή του προγράμματος περιήγησής σας και εκτελέστε:

```javascript
localStorage.removeItem("claude-initializr-config");
localStorage.removeItem("claude-initializr-welcome-dismissed");
localStorage.removeItem("claude-initializr-autosave");
```

Στη συνέχεια επαναφορτώστε τη σελίδα.

Εναλλακτικά, μπορείτε να απενεργοποιήσετε την αυτόματη αποθήκευση χρησιμοποιώντας τον διακόπτη στην κεφαλίδα για να αποτρέψετε την αποθήκευση ρυθμίσεων.
